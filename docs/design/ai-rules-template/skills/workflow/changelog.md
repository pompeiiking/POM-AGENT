# 变更日志规范

## 规则

- **每次变更必须在 CHANGELOG.md 中追加记录** — 无记录的变更视为未发生
- **CHANGELOG.md 是纯时间线** — 只追加不修改历史记录
- **每条记录必须包含五要素**：日期、操作者、任务、变更文件列表、影响范围
- **规范文档变更需额外溯源信息** — 见下方“规范文档变更的特殊要求”

## 格式模板

```markdown
### [YYYY-MM-DD HH:mm] — [操作者标识]

**任务**: [一句话描述本次变更的目标]
**变更文件**:
- `path/to/file1` — [具体改了什么]
- `path/to/file2` — [具体改了什么]

**影响范围**: [哪些模块/功能受影响]
**关联STUB**: [如有新增/替换的STUB，列出]
```

## 操作者标识

| 操作者 | 标识格式 | 示例 |
|--------|---------|------|
| 人类开发者 | 名字/代号 | `Alice`, `开发者A` |
| AI助手 | `AI:[工具名]` | `AI:Cascade`, `AI:Cursor` |
| 自动化工具 | `Bot:[工具名]` | `Bot:CI`, `Bot:Formatter` |

## 示例

✅ 正确：
```markdown
### 2026-03-04 14:30 — AI:Cascade

**任务**: 实现用户注册模块的表单验证
**变更文件**:
- `src/features/auth/RegisterForm.tsx` — 新增表单验证逻辑
- `src/features/auth/validators.ts` — 新建，注册字段校验规则
- `src/features/auth/types.ts` — 新增 RegisterFormData 类型

**影响范围**: auth模块
**关联STUB**: 无
```

❌ 错误：
```markdown
修改了一些文件
```
（缺少日期、操作者、具体内容、影响范围）

## 规范文档变更的特殊要求

以下文件属于**规范文档**，修改时影响所有开发者/AI的行为，必须额外记录溯源信息：

- `RULES.md`、`skills/` 下的所有skill文件
- `contracts/` 契约文档
- `STATUS.md` 的结构性修改（增删字段，而非常规状态更新）

### 规范文档变更记录模板

```markdown
### [YYYY-MM-DD HH:mm] — [操作者标识] — 📌规范变更

**任务**: [一句话描述为什么要改规范文档]
**变更文件**:
- `RULES.md:45` — [具体改了什么]

**变更前**:
```
[原文内容，逐字复制]
```

**变更后**:
```
[新内容，逐字复制]
```

**变更理由**: [为什么需要这个变更，解决什么问题]
**授权来源**: [人类开发者明确要求 / 完整性检查发现缺陷 / 其他]
**影响范围**: [哪些AI/开发者的行为会因此改变]
```

### 与普通代码变更的区别

| | 普通代码变更 | 规范文档变更 |
|--|------------|------------|
| 记录内容 | 文件列表 + 影响范围 | + 变更前/后对比 + 理由 + 授权来源 |
| 标识 | 无特殊标识 | 标题加 `📌规范变更` |
| 审查要求 | 常规 | 人类开发者应定期审查 |

## 不要放在CHANGELOG中的内容

以下信息属于 STATUS.md 而非 CHANGELOG.md：
- 项目当前进度
- 活跃的STUB清单
- 已知问题汇总
- 架构决策记录

## 提交前检查清单

- [ ] CHANGELOG.md 已追加本次变更记录
- [ ] 记录包含五要素：日期、操作者、任务、文件列表、影响范围
- [ ] 操作者标识使用了正确格式
- [ ] 变更文件列表完整，没有遗漏
- [ ] 若修改了规范文档，已使用规范变更模板（含变更前/后 + 理由 + 授权来源）
