# 项目状态管理（STATUS.md）

## 规则

- **STATUS.md 是项目当前状态的快照** — 不是历史记录，是"此刻"的真相
- **每次变更后覆盖更新**（不是追加），保持内容反映最新状态
- **新会话开始时必读** — 这是快速了解项目现状的入口
- **会话结束时必须更新** — 确保下一位开发者/AI获得准确信息

## STATUS.md 格式模板

```markdown
# [项目名] — 项目状态

> 最后更新: [YYYY-MM-DD HH:mm] by [操作者标识]

## 当前进度

[一两句话概括项目整体状态]

### 已完成模块
- [模块名] — [完成度/状态]

### 进行中
- [任务描述] — [当前状态/阻塞项]

### 待开始
- [任务描述] — [前置依赖]

## 活跃STUB清单

| 位置 | 原因 | 替换条件 |
|------|------|---------|
| `path/file:行号` | [为什么是STUB] | [何时可替换] |

> 如果清单为空，写"无活跃STUB"

## 已知问题

- [问题描述] — [严重程度: 高/中/低] — [是否阻塞当前开发]

## 架构备忘

[重要的架构决策、技术约束、需要所有开发者知道的信息]

## 下一步建议

[给下一位接手的开发者/AI的建议]
```

## 示例

✅ 正确的STATUS.md：
```markdown
# Pomtroller — 项目状态

> 最后更新: 2026-03-04 15:00 by AI:Cascade

## 当前进度

核心框架搭建完成，正在实现第一个功能模块（用户面板）。

### 已完成模块
- core/event-bus — ✅ 完成，已测试
- core/registry — ✅ 完成，已测试

### 进行中
- features/user-panel — 布局完成，数据绑定进行中

### 待开始
- features/settings — 依赖user-panel完成

## 活跃STUB清单

| 位置 | 原因 | 替换条件 |
|------|------|---------|
| `src/features/user/api.ts:42` | 后端API未就绪 | 后端完成用户接口后 |

## 已知问题

- 3D背景在Safari上帧率偏低 — 中 — 不阻塞当前开发

## 下一步建议

继续user-panel的数据绑定，完成后可开始settings模块。
```

## 提交前检查清单

- [ ] STATUS.md 已更新为当前最新状态（不是追加，是覆盖）
- [ ] "最后更新"时间和操作者已更新
- [ ] 活跃STUB清单与代码中的STUB标记一致
- [ ] 已知问题列表是当前的（已解决的问题已移除）
