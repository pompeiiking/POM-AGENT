# 长期记忆：保存内容与接口化定义

本文与代码 `src/core/memory/` 对齐，作为实现与评审的单一事实来源。

**架构细化（双库位置、会话内调用时序、Orchestrator）**：见 [`会话与双库长期记忆架构设计.md`](./会话与双库长期记忆架构设计.md)。

---

## 0. 当前状态（事实）

- **双库 + 嵌入（默认路径）**：由 `memory_policy.yaml` 的 **`dual_store_ref`** / **`embedding_ref`** 经 **`app/memory_orchestrator_registry.py`** 解析（与会话 `session_store_registry` 对称：`builtin:…` 或 `entrypoint:<name>`）。缺省为 `builtin:dual_sqlite` + `builtin:hash`；可选 **`builtin:openai_compatible`**（`/v1/embeddings`，密钥仅环境变量，参数见可选 **`embedding_openai`** 块）；库路径仍取自 `storage_profiles.memory.path`。
- **实现细节**：默认双库为 `SqliteDualMemoryStore`（`memory_items` + `memory_fts` + `memory_vectors`）+ `MemoryOrchestrator`；向量检索为 **余弦扫描**（受 `vector_max_candidates` 限制），非独立 HNSW 服务。
- **配置开关**：`platform_layer/resources/config/memory_policy.yaml` 中 `enabled: false` 时**不**构建 Orchestrator，长期记忆与 `search_memory` 工具为空操作语义。
- **关卡⑤（资源访问）**：`resource_access.yaml` 中 `long_term_memory` 的 `read`/`write` 控制上下文注入、`/remember`/`/forget`、归档晋升写入与 `search_memory`（见 `core/resource_access.py`）。
- **历史协议**：`LongTermMemoryStore` / `memory_store_registry` / `sqlite_long_term_memory_store` 仍保留作**插件/兼容**，主链路以 Orchestrator 为准。

## 1. 长期记忆的保存内容（What）

长期记忆与**会话存储（SessionStore）**分离：会话存「当前对话状态」；长期记忆存**跨会话、可检索、可延续**的知识与引用。

### 1.1 记忆块（Memory Chunk）— 检索主单元

面向 **RAG / 混合检索** 的基本写入单位。

| 字段 | 说明 |
|------|------|
| `text` | 参与检索的正文（多模态入长期记忆时须先降为文本或摘要） |
| `user_id` | 归属用户（分区键） |
| `channel` | 可选；若策略要求按频道隔离记忆则填写 |
| `source_session_id` / `source_message_id` | 可选，追溯来源 |
| `trust` | 信任等级：`low` / `medium` / `high`，供组装时打标签与关卡⑤ |
| `tags` | 可选，便于过滤 |
| `created_at` | 写入时间（可由存储生成） |

**典型来源**（设计层面，待管理编排落地后）：归档会话分段、工具结果摘要、显式「请记住」类指令经解析后的文本。

### 1.2 用户偏好（User Preference）

跨会话稳定的**键值或短文本**，用于注入系统/记忆区（非全文检索主路径也可单独读取）。

| 字段 | 说明 |
|------|------|
| `user_id` | 归属用户 |
| `key` | 命名空间下的键（如 `locale`、`tone`） |
| `value` | 字符串或 JSON 字符串 |
| `updated_at` | 更新时间 |

### 1.3 事实陈述（Fact）

从对话或工具中抽取的**可验证陈述**，便于冲突处理与高精度召回。

| 字段 | 说明 |
|------|------|
| `user_id` | 归属用户 |
| `statement` | 自然语言或规范化陈述 |
| `confidence` | 0.0–1.0 |
| `evidence_ref` | 可选，指向 `session_id` / `message_id` / chunk id |

### 1.4 归档链接（Archive Link）

指向**已归档会话**的索引项，便于「按会话维度」召回，而不重复存全量 messages。

| 字段 | 说明 |
|------|------|
| `user_id` | 归属用户 |
| `session_id` | 已归档会话 id |
| `archived_at` | 归档时间 |
| `summary_excerpt` | 用于检索/展示的短摘要（可与 `session_archives.summary_text` 对齐或截取） |

### 1.5 与现有表的关系

- **`sessions` / `session_archives`**：仍属**会话域**；长期记忆可实现为**另一库/另一套表**，通过 `ArchiveLink` 与归档行关联，避免混为一张大表。
- **`storage_profiles.memory`**：**`path`** 为双库 SQLite 文件路径（`MemoryOrchestrator` 使用）；**`store_ref`** 仅对应旧版 `LongTermMemoryStore` 注册表。`memory_policy.enabled=true` 时 **`store_ref` 必须为 `builtin:noop`**（由 `resource_validation` 校验），主线不由 `LongTermMemoryStore` 写同一文件。

## 2. 长期记忆的接口化（How to plug）

### 2.1 原则（与主线对齐）

- **主线**：`MemoryOrchestrator` + `DualMemoryStore`，由 `memory_policy.yaml` 的 `dual_store_ref` / `embedding_ref` 经 `app/memory_orchestrator_registry.py` 解析。
- **`LongTermMemoryStore`**：旧协议 + `memory_store_registry`，**不接入** `composition.build_core`；仅供插件/测试或将来独立实验。
- **配置约束**：`memory_policy.enabled=true` ⇒ `storage_profiles.memory.store_ref` **必须** `builtin:noop`，避免与双库共路径冲突。

### 2.2 协议能力（最小集）

1. **`put(record)`**：写入或更新一条长期记忆记录（联合类型见 `content.py`），返回稳定 `record_id`。
2. **`search(query)`**：按 `user_id` + 查询文本返回有序命中列表（向量/BM25/混合由实现决定）。
3. **`delete_user_data(user_id)`**（可选但推荐）：合规删除该用户相关长期记忆。

嵌入生成（`model.embed`）、切块策略、异步队列均属**实现细节**，不进入核心协议。

### 2.3 代码位置

| 路径 | 内容 |
|------|------|
| `src/core/memory/content.py` | 保存内容类型（dataclass） |
| `src/core/memory/protocol.py` | `LongTermMemoryStore`（旧线）与查询/结果类型 |
| `src/app/memory_store_registry.py` | 旧线热插拔（**主装配未用**） |
| `src/app/memory_orchestrator_registry.py` | 双库 + 嵌入（**主线**） |
| `src/infra/sqlite_long_term_memory_store.py` | 旧线示例子串 SQLite（**非向量**） |
| `src/core/memory/__init__.py` | 导出公共 API |

### 2.4 `storage_profiles.memory` 段

- **`path`**：`MemoryOrchestrator` / `DualMemoryStore` 的 SQLite 路径（主线）。
- **`store_ref`**：仅用于 **`LongTermMemoryStore` 旧线**。主线开启记忆时须 **`builtin:noop`**；若 `memory_policy.enabled=false`，可保留 `noop` 或仅作占位；**勿**在开启 Orchestrator 的同时设为 `builtin:sqlite` 指向同一 `path`（校验会拒绝）。
- 旧线可选值（插件/测试）：`builtin:noop`、`builtin:sqlite`、`entrypoint:<name>`（`pompeii_agent.long_term_memory_stores`）。

## 3. 管理逻辑（会话域 → 长期记忆）— **待设计**

以下仅列**问题域**，**不在实现中假定答案**：

- **触发条件**：仅归档？显式指令？定时/空闲？会话条数/容量？
- **内容选择**：全量摘要、尾部窗口、LLM 摘要、结构化事实抽取？
- **与合规**：用户同意、删除策略、与 `session_archives` 的去重关系。
- **与向量管线**：何时 embed、索引更新频率、失败重试。

**待设计评审通过后再**实现独立编排模块（或显式扩展点），**不得**在核心 Loop 内隐式写入长期记忆。
