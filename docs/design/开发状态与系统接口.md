## 3. 系统分层与状态

### 3.1 运行时分层

```
platform_layer/resources/config/*.yaml
        ↓
app/composition.build_core / build_port  （组合根）
        ↓
port/GenericAgentPort  （唯一交互边界：PortInput → Core → PortEvent）
        ↓
core/AgentCoreImpl     （会话 + loop + 策略）
        ↓
modules/assembly | model | tools
        ↓
infra/  （SQLite、MCP 桥接等）
```

- **`core` 不依赖 `app`/`port`**，便于单测与替换存储。  
- **`port` 依赖 `core` 类型**，负责把「原始输入」变成 `AgentRequest`，并把 `AgentResponse` 变成 `PortEvent`。

### 3.2 状态存放（系统状态）

| 状态类型 | 存放位置 | 说明 |
|----------|----------|------|
| 会话消息与配置 | `SqliteSessionStore`（文件或 `ephemeral()`） | 权威对话历史 |
| 待工具确认 | `GenericAgentPort` 内 `_pending_confirmation` | **进程内**；按 `(user_id, channel)` |
| 待设备结果 | `GenericAgentPort` 内 `_pending_device_request` | 同上 |
| 归档摘要行 | SQLite `session_archives`（同一存储实现） | 规则生成摘要 |

**多进程 / 多 worker**：待定字典不跨进程；需外置存储或粘性会话才能水平扩展。

## 4. 整体功能流（深度）

### 4.1 主路径：普通用户消息

1. **入口**：CLI 一行文本，或 HTTP `POST /input`（`kind: user_message`）。  
2. **Port**：`UserMessageInput` → `RequestFactory`（CLI 固定 `request_id`；带 `user_id/channel` 时用 `session_request_factory` 生成 UUID `request_id`）→ `AgentRequest`。  
3. **Core.handle**：  
   - `get_or_create_session`  
   - **追加 user 消息**  
   - `assembly.build_initial_context`  
   - `_run_loop`：  
     - `model.run(session, context)` → `ModelOutput`  
     - **text** → `assembly.format_final_reply` → 写 assistant → `AgentResponse(ok)`  
     - **tool_call** → 策略：白名单 / 确认 / 设备映射 → `execute` 或 `device_request` 或 `confirmation_required`  
4. **Port**：`AgentResponse` → `_handle_core_response` → `ReplyEvent` / `ConfirmationEvent` / `DeviceRequestEvent` / `ErrorEvent` 等 → `emit`。

### 4.2 子路径：工具确认后执行

1. 用户在下一次输入中提交确认（CLI `yes`/`/confirm …`；HTTP `system_command`）。  
2. Port 命中 `_pending_confirmation`，调用 **`core.handle_confirmation_approved(request, tool_call)`**（**不重复追加 user**）。  
3. 与主路径汇合：策略 → 执行工具 → `assembly.apply_tool_result` → 继续 loop。

### 4.3 子路径：设备结果

1. 先前返回 `device_request`。  
2. Port 存 `_pending_device_request`；用户回传 `device_result`（CLI/HTTP）。  
3. **`core.handle_device_result`**：写 tool 消息 → `_run_loop`。

### 4.4 子路径：系统命令与归档

- **`/archive` 等**：`AgentRequest.intent` 为 `SystemArchive` → `_handle_archive_command` → 会话 `ARCHIVED` → 存储层写归档行（若后端支持）。

### 4.5 数据流小结

- **向下**：原始输入 → 结构化请求 → 会话追加 → 上下文视图 → 模型/工具。  
- **向上**：模型/工具结果 → 会话追加（OpenAI v1 形态 assistant/tool）→ 响应 → 端口事件。

## 5. 功能完成度与边界

### 5.1 已具备的整体能力

- 多轮对话 + 工具闭环 + **确认流** + **设备请求流**（逻辑完整）。  
- **MCP stdio**（可选）扩展工具来源。  
- **会话持久化**与**归档列表**（统一 SQLite 实现）。  
- **组装部预算**：条数尾部、单条字符、近似 token 总量。

### 5.2 与设计稿 ver0.4 差距（摘要）

- **向量 / RAG / 长期画像**：长期记忆双库与检索已部分落地；用户画像、GraphRAG 等仍缺。  
- **资源区守门**：内核工具白名单/确认 + **`resource_access.yaml`**（关卡⑤：`long_term_memory` 读/写、`multimodal_image_url`、配置键校验）；尚无通用资源目录与高敏「核心审批」流。  
- **关卡④ 结果净化**：**部分**——工具回流已有长度/守卫、**来源分区**（`tool_result`）、**默认注入子串**（伪造 `pompeii` 分区边界 + JSON 视图扫描）；广谱注入词库、低信任守卫模型审核、沙箱 ④-d 仍弱。  
- **LLM 归档摘要**：可选 **`kernel.archive_llm_summary_*`** 异步写入 `session_archives`；默认展示仍为规则摘要。

## 7. 系统接口清单

### 7.1 HTTP（实验性，FastAPI）

| 方法 | 路径 | 作用 |
|------|------|------|
| GET | `/health` | `ok` + `version` |
| GET | `/archives` | 查询用户归档列表（`user_id` 必填） |
| POST | `/input` | `kind`: `user_message` / `system_command` / `device_result`；返回 `events` 数组（可含 `delegate`、`reply`、`confirmation` 等） |

**开放程度**：**中等**——接口稳定但属实验形态；未提供 OpenAPI 之外的正式 API 版本策略。

### 7.2 内核入口（`AgentCore` / `AgentCoreImpl`）

| 接口 | 作用 |
|------|------|
| `handle(AgentRequest)` | 主入口：追加 user → loop |
| `handle_confirmation_approved(AgentRequest, ToolCall)` | 确认后执行工具并续跑 |
| `handle_device_result(..., ToolResult, tool_call_id?)` | 设备结果回注并续跑 |
| `list_archives_for_user(user_id, limit?)` | 归档列表 |

**开放程度**：**高**——面向组合根与测试，类型清晰。

### 7.3 Port 边界

| 类型 | 作用 |
|------|------|
| `GenericAgentPort.handle(PortInput, user_id?, channel?, emitter?)` | 统一入口 |
| `PortEmitter.emit` / `HttpEmitter.dump` | 事件收集与输出 |

### 7.4 模块 Protocol（可替换契约）

| Protocol | 位置 | 职责 |
|----------|------|------|
| `AssemblyModule` | `modules/assembly/interface.py` | 构建上下文、工具回注、最终回复格式化 |
| `ModelModule` | `modules/model/interface.py` | `run` → `ModelOutput` |
| `ToolModule` | `modules/tools/interface.py` | `execute` → `ToolResult` |
| `DeviceToolBackend` | `modules/tools/device_backend.py` | 可选本地设备桩 |
| `SessionStore` | `core/session/session_store.py` | 会话 CRUD、归档列表 |
| `SessionManager` | `core/session/session_manager.py` | 会话生命周期 |
| `InteractionMode` | `port/agent_port.py` | CLI 循环 vs HTTP 占位 |
| `PortEmitter` | `port/agent_port.py` | 事件输出 |

### 7.5 配置文件（对外行为的主要旋钮）

| 文件 | 内容 |
|------|------|
| `session_defaults.yaml` | 会话模型、limits（含组装预算） |
| `kernel_config.yaml` | `max_loops`、工具白名单、需确认列表、可选 `delegate_target_allowlist` |
| `model_providers.yaml` | 模型 provider、backend、环境变量名等 |
| `runtime.yaml` | 仅 **`sqlite`** + `sqlite_path`（测试：`SqliteSessionStore.ephemeral()`） |
| `mcp_servers.yaml` | MCP 启用与进程 |

## 8. 开放程度与可替换模块

### 8.1 开放程度（定性）

| 层次 | 评价 | 说明 |
|------|------|------|
| **内核 core** | 较高 | 策略拆分、`ToolDeps`、可单测；与具体 HTTP 框架无关 |
| **模块 modules** | 高 | 三模块均为 Protocol + 默认 Impl；可并行实现替换 |
| **存储** | 高 | `SessionStore` 抽象；当前 **`SqliteSessionStore`**；JSON codec 可扩展字段 |
| **组合根 app** | 中 | `build_core` 集中 wiring；新运行时需抄或抽工厂 |
| **Port HTTP** | 中 | 与 FastAPI 绑定；事件 DTO 可视为稳定契约 |

### 8.2 可替换模块（推荐替换点）

| 组件 | 替换方式 | 备注 |
|------|----------|------|
| **模型后端** | 实现 `ModelModule` 或替换 `ModelModuleImpl` | 已支持 registry |
| **工具来源** | 扩展 `ToolModuleImpl`、接 MCP、或换 `ToolModule` | MCP 已桥接 |
| **组装策略** | 替换 `AssemblyModuleImpl` | 预算逻辑已模块化 |
| **会话存储** | 实现 `SessionStore` | 需保持 JSON codec 或同步更新 |
| **会话统计** | `session_store_ops.append_message_inplace` 已统一一处 | 扩展统计字段时改此与 `SessionStats` |
| **设备本地桩** | `DeviceToolBackend` | 与 Port 设备流并存 |
| **意图解析** | `port/intent_parser.py` | 影响 `AgentRequest.intent` |

### 8.3 耦合点（替换时需注意）

- **`ModelOutput` / `ToolCall.call_id`**：与 `openai_message_format`、core 写消息强相关。  
- **`KernelConfig` 路径**：`composition` 固定读 `platform_layer/.../kernel_config.yaml`。  
- **HTTP 与 Port 共享单例 `_HTTP_PORT`**：扩展路由时勿绕过 Port 状态机。

## 9. 结论与建议

- **当前系统**：在「单进程、演示/轻量生产」前提下，**端到端功能流完整**，模块边界与测试覆盖优于典型脚本型 Agent。  
- **主要缺口**：水平扩展、流式模型、向量与资源治理、LLM 归档摘要。  
- **对接建议**：第三方集成优先走 **`Port` + `AgentCore` 公共类型**；避免复制 `session.messages` 写逻辑到应用层。
